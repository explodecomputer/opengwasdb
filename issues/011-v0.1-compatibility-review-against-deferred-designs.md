## What to build

Review the completed v0.1 Dense Observed-Only implementation against the spec and ADRs for deferred designs. The goal is to catch Dense-only assumptions before they become hard to unwind.

## Acceptance criteria

- [ ] Public query API does not expose Dense-only concepts that would block Ragged adapters.
- [ ] Build pipeline keeps ingestion/normalisation separate from Dense writing.
- [ ] Store manifest and validation leave room for Reference-Completed releases.
- [ ] Reference Completion extension points can record the `pleiodb` imputation method, LD block TSV files, `.unphased.vcor1.gz` LD matrices, optional `.ldeig.rds` eigenfactor files, and cached eigenfactor extracts.
- [ ] Result contract can represent future Association Status and Query Component fields.
- [ ] Any identified incompatibilities are documented as follow-up issues or resolved before v0.1 is considered complete.
- [ ] Review notes explicitly reference relevant spec and ADR sections.
